Apache Derby Guide

Set Environment variables

DERBY_INSTALL
C:\ApacheDerby
CLASSPATH
%DERBY_INSTALL%\lib\derbytools.jar;%DERBY_INSTALL%\lib\derbynet.jar;%DERBY_INSTALL%\lib\derbyclient.jar;


Edit config files 
(as far as I know by time of writing 08.03.2020)
Guide and config files were verified by install from zero on 25-08-2020

1.	Server.policy file
a.	Located in {derby_install_directory}/demo/templates/server.policy
2.	Java.policy file
a.	Located in {jre_install_directory} (NOT IN JDK install directory)/lib/security/java.policy

Run Database Server Instance
1.	open cmd line(you don't have to be admin).
2.	Start server by issuing following command
java -Djava.security.manager -Djava.security.policy=C:/dbDerby/demo/templates/server.policy  org.apache.derby.drda.NetworkServerControl start -h localhost

Run Client connection from cmd
1.	Open another cmd
2.	Run client from cmd by issuing following command:
java org.apache.derby.tools.ij
3.	Connect to a database MyDbTest (if it does not exist create it):
connect 'jdbc:derby://localhost:1527/C:/dbDerby/demo/databases/MyDbTest;create=true';


Exit ij console
1.	If you connected to server:
disconnect;
exit;
2.	If you not connected to server:
exit;

In SQuirell client
 if you see following error message
ERROR: see the next exception for details.::SQLSTATE: XBM0W
1.	Check environmental variables
2.	Start client from cmd line
